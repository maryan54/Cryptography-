1. INTRODUCTION
================================================
Secure communication over the Internet depends on
the ability of two parties to exchange information
privately, even over an insecure network. Before
1976, this required both parties to share a secret
key in advance, which was impractical for large-scale
use. The Diffie-Hellman Key Exchange protocol,
developed by Whitfield Diffie and Martin Hellman
in 1976, solved this problem by allowing two parties
to establish a shared secret key over a public
channel without ever transmitting the key itself.
This report examines how the protocol works, the
mathematical problem that secures it, its
applications in modern internet security, and
the challenges it faces today.

================================================
2. HOW DIFFIE-HELLMAN WORKS
================================================
Diffie-Hellman works using modular arithmetic
and agreed public parameters.

Step 1 - Public Parameters:
Both parties agree on a large prime number (p)
and a generator (g). These values are public.
Example: p = 23, g = 5

Step 2 - Private Keys:
Alice chooses private key: a = 6
Bob chooses private key:   b = 15

Step 3 - Generate Public Keys:
Alice computes: A = g^a mod p = 5^6 mod 23 = 8
Bob computes:   B = g^b mod p = 5^15 mod 23 = 19

Step 4 - Exchange Public Keys:
Alice sends A = 8 to Bob (publicly)
Bob sends B = 19 to Alice (publicly)

Step 5 - Compute Shared Secret:
Alice computes: S = B^a mod p = 19^6 mod 23 = 2
Bob computes:   S = A^b mod p = 8^15 mod 23 = 2

Both arrive at the same secret key (2) without
ever transmitting it. An attacker watching the
exchange only sees p, g, A, and B, not the
private keys or the secret.

================================================
3. THE DISCRETE LOGARITHM PROBLEM
================================================
The security of Diffie-Hellman is based on the
Discrete Logarithm Problem (DLP). Given:

   g^a mod p = A

It is easy to calculate A when you know a, g,
and p. However, given only A, g, and p, finding
the private key (a) is extremely difficult when
p is a large prime number (2048 bits or more).

This one-way mathematical property is what makes
Diffie-Hellman secure. Even with powerful computers,
solving the discrete logarithm for large primes
would take millions of years using current
algorithms. This is an intractable problem, easy
to compute in one direction and practically
impossible to reverse.

================================================
4. APPLICATIONS IN TLS/SSL
================================================
Diffie-Hellman is widely used in TLS (Transport
Layer Security), the protocol that secures
HTTPS connections.

TLS Handshake using Diffie-Hellman:
1. Client connects to server (https://www.bank.com)
2. Server sends its digital certificate
3. Client verifies the certificate via PKI
4. Both parties perform Diffie-Hellman key exchange
5. A shared session key is derived
6. All communication is encrypted using that key

The modern version used is ECDHE (Elliptic Curve
Diffie-Hellman Ephemeral), which provides:

- Perfect Forward Secrecy (PFS): A new key is
  generated for every session, so past sessions
  cannot be decrypted even if the private key
  is later compromised.

- Stronger security with smaller key sizes using
  Elliptic Curve mathematics.

Other applications include:
- VPNs (IPSec and OpenVPN)
- SSH (Secure Shell) connections
- Signal and WhatsApp encrypted messaging
- Email encryption (PGP/GPG)

================================================
5. SECURITY CHALLENGES AND SOLUTIONS
================================================
Challenge 1 - Man-in-the-Middle Attack:
Diffie-Hellman alone does not provide
authentication. An attacker can intercept the
exchange and impersonate both parties.
Solution: Combine with PKI digital certificates
to authenticate both parties before key exchange.

Challenge 2 - Weak Parameters:
Small prime numbers make the discrete logarithm
solvable. The Logjam attack (2015) exploited
weak 512-bit DH parameters in many servers.
Solution: Use a minimum of 2048-bit primes.
Modern systems use 3072 or 4096-bit parameters.

Challenge 3 - Quantum Computing Threat:
Quantum computers running Shor's algorithm could
theoretically solve the discrete logarithm
problem, breaking standard Diffie-Hellman.
Solution: Post-Quantum Cryptography algorithms
are being developed. NIST standardized several
quantum-resistant algorithms in 2024.

Challenge 4 - Static vs Ephemeral Keys:
Static DH keys reuse the same parameters,
meaning if compromised, all past sessions
can be decrypted.
Solution: Use Ephemeral Diffie-Hellman (ECDHE)
which generates a fresh key pair every session.

================================================
6. CONCLUSION
================================================
The Diffie-Hellman Key Exchange protocol
revolutionized cryptography by solving the
fundamental problem of secure key distribution
over public networks. Its mathematical foundation,
the Discrete Logarithm Problem, has proven
remarkably robust for nearly five decades. Today,
in its modern form as ECDHE, it remains the
primary mechanism for establishing secure sessions
in HTTPS, VPNs, and encrypted messaging apps.
While threats from quantum computing loom on the
horizon, the principles introduced by Diffie and
Hellman continue to underpin the security of the
modern Internet.
